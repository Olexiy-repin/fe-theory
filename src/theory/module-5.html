Позиціювання елементів

Браузер зображує на сторінці вміст документу html, наслідуючи потік виконання документу. Для європейських мов потік подібний до правил читання: зверху вниз, зліва направо.

Як буде, якщо елемент розташований над межею двох блоків? Як змусити меню залишатися завжди на одному місці сторінки, тимчасом як решту вмісту переміщувати з допомогою гортання? Для виконання подібних завдань було створено властивість position.

Ця властивість немов підіймає елемент над потоком виконання і розміщує елемент над будь-яким елементом, що міститься в потоці. Що буде, якщо два такі елементи поза потоком опиняться в одному місці, хто буде в тіні, а хто на вершині? Цю проблему вирішує інша властивість - z-index.

position

Вказує особливі правила розміщення елементу. Зокрема для визначення положення елементу використовуються властивості: top, left, bottom або right, без position вони не застосуються.

Відносне позиціювання

Якщо position: relative, елемент залишається в потоці, його розміри та положення обраховуються для усіх сусідніх блоків відповідно до box-model. Промальовування зображення робиться браузером з урахуванням вказаних властивостей: top, left, bottom або right. Водночас може відбуватися перекриття зображення сусідніх елементів.

Абсолютне позиціювання

position: absolute - підіймає елемент над потоком, тобто після застосування цієї властивості сусідні елементи не враховуватимуть цей елемент у розрахунку для визначення свого положення.

Відлік своїх top, left, bottom або right для absolute-елементу ведеться відносно найближчого предка з властивістю, відмінною від властивості position: static. Якщо такого предка не буде, то відлік робитиметься від body.

Коли до елементу додається властивість position: absolute, то завжди слід відповісти на питання: "Який елемент буде тим предком, до якого потрібно буде застосувати властивість position: relative ?". Після відповіді на це питання необхідно переконатися, що position: relative встановлено.

При розрахунку положення елементу в потоці треба було враховувати тільки верхній лівий кут елементу і батьківського контейнера, а відлік положення елементу, що абсолютно позиціюється, інший.

Запис bottom: 20px означає відстань між нижньою межею батьківського контейнера і нижньою межею самого елементу. Аналогічно працюють right, left top: ними задається відстань між однойменними сторонами контейнера та елементу.

  • right: 50px - права межа елементу на 50px від правого краю контейнера
  • top: 50px - верхня межа елементу на 50px від верхнього краю контейнера
  • bottom: 50px - нижня межа елементу на 50px від нижнього краю контейнера
  • left: 50px - ліва межа елементу на 50px від лівого краю контейнера

Приклад, що демонструє застосування position: absolute. Чорна точка вказує на початок відліку, тобто верхню ліву точку першого батьківського контейнера з position: relative. Блок по центру відцентрований з допомогою absolute і transform: translate. Інші блоки показують зміщення залежно від властивостей і значень.

Властивість transform і його значення translate(translateX, translateY) зміщують елемент відносно його поточного положення. translateX зміщує верхній лівий кут елементу по горизонталі, а translateY - по вертикалі. Розрахунок положення як і в box-model. аке дублювання способу переміщення потрібне, щоб за допомогою властивості transform можно здійснювати анімацію.

See the Pen position_absolute by GoIT ( @GoIT-fe) on CodePen.

Не варто забувати, що на елементи з position продовжують діяти правила, за якими ці елементи зміщуватимуться при margin і змінюватимуть розмір при padding. Ця особливість дозволяє центрувати елемент по вертикалі контейнера.

Для цього зміщуємо лівий верхній край по центру контейнера, а потім зміщуємо увесь елемент вліво і вгору на половину ширини та висоти самого елементу. 

element {
  position: absolute;
  left: 50%;
  right: 50%;
  transform: translateX(-50%) translateY(-50%);
}

Позиціювання з фіксацією положення

position:fixed це аналог значення absolute. Відмінність полягає в точці відліку. Для fixed відлік завжди відносно меж вікна браузеру. Це створює ефект фіксації елементу при використанні вертикальної прокрутки. Зазвичай властивість position: fixed використовують для елементів навігації, що дозволяють перейти до початку сторінки, або для верхнього меню, як в прикладі на зображенні.

Для демонстрації роботи position fixed наведіть курсор під написом "I am fixed" і використайте колесо миші для прокрутки контенту, оскільки scrollbar цього прикладу був штучно схований.

"Липке" позиціювання

position: sticky це комбінація властивостей relative і fixed. Поки елемент міститься у своєму контейнері й контейнер не покидає область екрану при вертикальній прокрутці, це relative. Якщо частина батьківського контейнера покинула видиму частину екрану, то властивість змінюється на fixed, тобто фіксується на сторінці, поки частина контейнера є на екрані. Якщо увесь контейнер покине зону видимості, то знову спрацює властивість relative, тобто зображення елементу зникне з екрана услід за контейнером.

Скасування дії позиціювання

position: static це звичайне (усталене) значення властивості position усіх елементів. Його можна змінити, вказавши інше значення, а потім скасувати дію relative, absolute або fixed присвоївши знову значення static.

Властивість z-index

До властивості position ми не надавали великого значення розташуванню елементів по осі z (перпендикуляру до площини екрану). Розташування елементів у цій площині було інтуїтивно зрозуміле. Предок розташовувався внизу, подібно до аркуша паперу, а нащадок — немов новий аркуш паперу, який лягає згори. Що більше нащадків було в елементу, то більше було таких шарів.

Застосування position все змінило. Елементи з position, відмінним від static, випадають з потоку, а значить для таких елементів треба правило, яке визначатиме який елемент знаходиться вище, а який нижче. Це правило задає властивість z-index, значеннями якого може бути будь-яке ціле число, зокрема від'ємне.

Правила застосування

Загальний предок створює контекст, область в якій z-index елементів робить вплив на інші елементи з z-index. Усередині цього контексту значення z-index може бути більше інших, і тоді елемент розташовується згори.

Мінімальне (навіть від'ємне) значення z-index говорить про те, що інші елементи поза потоком будуть вище цього елементу. У разі рівності значення z-index (це значення усталено) вище буде елемент, оголошений останнім.

Винятком з правил може стати ситуація, коли якийсь з предків без властивості z-index отримає напівпрозорий фон, тобто його opacity стане відмінною від 1. У такому разі відбувається формування нового контексту і подібних випадків краще уникати у зв'язку з неочевидністю застосування правил через несподівану появу нового загального предка.

Верстання випадного меню

Випадне меню — це найбільш популярний тип меню на вебсторінках. Розгляньмо етапи створення такого меню. Запропоновані імена класів приведені для ясності викладу.

  1. Необхідно створити звичайне меню, присвоївши class="menu" блоку і class="menu-item" елементам меню.
  2. Усередині menu-item розмістите нове меню, з class="submenu" і елементами class="submenu-item".
  3. Приховуємо вкладене меню
    .submenu { display: none; }
  4. Задаємо правило появи вкладеного меню при наведенні курсора
    .menu-item:hover > .submenu { display: block; }
  5. З допомогою position: absolute розміщуємо submenu відносно li так, щоб була можливість перейти до пунктів submenu.

Ключовим є правило появи вкладеного меню. Це правило працює так: при наведенні на елемент меню, що містить вкладене меню, спрацює псевдоклас hover і прямий нащадок (на це вказує знак >) стане видимим.

Також дуже важливий останній етап. Наприклад, за наявності 1px межі (border) у menu може відбуватися скасування дії hover. Річ у тому, що на сам блок menu ефект hover не поширюється. Його дія пов'язана тільки з menu-item і якщо браузер зафіксує курсор не над menu-item, то повернеться усталений стан і вкладене меню зникне. Таку помилку складно виявити, оскільки браузер не відстежує кожен пройдений курсором піксель, і помилка проявляється час від часу.

Стандарт HTML5

Семантичне верстання призначене для опису смислової структури документу. Саме для розділення контенту на різні за типом і змістом частини: заголовки, абзаци, зображення, посилання тощо.

До HTML5 частина тегів містила мінімальне смислове навантаження "контейнер" - div і span. Частина тегів взагалі мала відношення не до змісту, а до зовнішнього вигляду - br, hr, i, b тощо. HTML5 надав для структуризації й угрупування контенту нові елементи, які здатні передати семантику блоку. Наприклад, header і footer.

З практичної точки зору подібні зміни полегшують аналіз вмісту вебсторінок пошуковим роботам, програмам для людей з обмеженими можливостями тощо.

За смисловим змістом усього виділяють 7 груп елементів :

  1. Мета вміст
  2. Потоковий вміст
  3. Секційний вміст
  4. Заголовний вміст
  5. Текстовий вміст
  6. Вбудований вміст
  7. Інтерактивний вміст

Структурні елементи

Тоді як div може використовуватися для представлення нижчевказаних розділів, використання семантичних елементів забезпечує багатший і вагоміший документ.

header

Заголовок сторінки або розділу, що містить ввідний контент або навігацію. Не є обов'язковим. У HTML документі може міститися одночасно декілька елементів header, і вони можуть розташовуватися у будь-якій частині сторінки.

header не можна поміщати всередину елементів: footer, address, header

nav

Призначений для створення блоку навігації вебсторінки або усього вебсайту. Блок навігації — основна група посилань на інші сторінки або розділи сторінки. На сторінці може бути декілька елементів. Не замінює теги ul або ol, він просто їх огортає, у такий спосіб надаючи семантичне значення.

Не усі групи посилань повинні огортатися nav, цей елемент призначений для розділів, які складаються з головних навігаційних блоків.

article

Розділ, що цілком містить деяку тему. Незалежна частина документу або сайту. Використовується для угрупування записів: публікацій, статей, записів блогу. Зазвичай, розпочинається із заголовка.

section

Загальний розділ документу. Може використовуватися для розділення статті або, наприклад, для представлення глав. Це універсальний розділ документу. Групує тематичний вміст і зазвичай містить заголовок.

Не є блоком-обгорткою, для таких цілей використайте елемент div.

main

Основний зміст body - унікальний для сторінки, виключаючи контент, який може повторюватися на декількох сторінках, наприклад, навігація.

Не може бути вкладений в елементи:: article, aside, footer, header або nav

aside

Групує вміст, пов'язаний з контентом, що оточує його. Розділ, розташований збоку від основного, містить менш вагому інформацію (рекламні блоки, архів тощо).

Не підходить для блоків, що просто позиціюються осторонь.

footer

Нижній колонтитул сторінки або розділу. Може містити контент, наприклад, постскриптум, додаток або посилання на пов'язані сторінки, дані про копірайт, контактну інформацію.

У документі може бути декілька елементів footer. Як кожна сторінка, так і кожна стаття може мати свій footer.

figure и figcaption

Автономний ілюстративний контент, який зазвичай згадується в основному змісті сторінки. Використовується для додавання опису до фотографій, діаграм, фрагментів коду. figcaption — нащадок figure. Використовується як контейнер для утримання описової частини зображення.

Елементи семантики тексту

  • address - для зображення контактних даних власника документу або статті.
  • time - визначає час або дату. Для тега доступний атрибут datetime, його значення бачитиме користувач на екрані свого комп'ютера.
  • mark - виділяє фрагменти тексту, позначаючи їх жовтим фоном, зазвичай при збігу з текстом в рядку пошуку.
  • small - коментарі - зображає текст шрифтом меншого розміру.
  • cite - для вказівки джерела цитування.
  • wbr - можливе місце розриву довгого рядка.
  • bdi, bdo - для тексту, написаного на мові з потоком, відмінним від потоку документу.
  • ruby, rb, rt, rtc, rp - для роботи зі східни ми мовами.
An Overview of HTML5 Semantics

HTML5 media

У HTML5 використання потокового медіаконтенту стало стандартним і не вимагає використання сторонніх інструментів. Розглянемо як працювати з потоковим відео і звуком.

audio

Парний тег, атрибути якого визначають усі його можливості:

  • src - визначає джерело звуку або набір джерел в різних форматах.
  • controls - елементи управління прослуховуванням, визначаються браузером.
  • loop - якщо цей атрибут вказаний, то відтворення повторюватиметься знову і знову.
  • preload - вказує спосіб кешування:
    • none: не завантажує нічого до запуску користувачем.
    • metadata: завантажує тільки назву треку, тривалість і подібні дані, але не звук.
    • auto: завантажує все.

Крім того, тег source може бути вкладений в тег audio і містити атрибути src для потоків з різними кодеками.

Підтримка кодеків ogg, mp3, wav. Щоб запобігти ситуації, коли браузер користувача не підтримує запропоновані вами кодеки, між тегами audio розміщують текст-попередження.

video

Парний тег, що має аналогічні з audio атрибути, а також додаткові:

  1. width і height вказують розміри для зображення відеопотоку.
  2. autoplay автоматичний запуск програвання.
  3. poster посилання на зображення, яке стане фоном виділеного width і height простору до запуску відеопотоку

Підтримка кодеків ogv, mp4, flv, webm

Крім того, тег track може бути вкладений в тег video і містити атрибути для субтитрів kind="subtitles" src="boom.en.vtt" srclang="en" label="English" або інші дані. Цей тег також може використовуватися для audio.

Змінні CSS

Змінні в CSS на даний момент підтримуються всіма основними браузерами за винятком часткової підтримки Edge і не підтримуються IE і Opera Mini. Проте, ця технологія обов'язково буде використовуватися в самий найближчий час і варто розповісти про її можливості.

На відміну від змінних в препроцесорів CSS (розповідь про них буде пізніше), змінні CSS залишаються змінними і під час виконання програми, в той час як змінні препроцесорів після компіляції в CSS стають звичайними значеннями. Цей факт дуже зручний для створення JavaScript додатків.

Досить просто створювати інтерактивність при оформленні сторінки (користувач може змінювати будь-які параметри CSS, якщо надати йому таку можливість). Але основне застосування змінних в CSS це скорочення обсягу коду і спрощення його підтримки.

Для оголошення змінної перед властивістю досить вказати два дефіса, а для використання замість значення написати var() і всередині круглих дужок помістити ім'я змінної з двома дефісами попереду. 

element { --main-bg-color: teal; }

element { background-color: var(--main-bg-color); }

Проблема в тому, що оголошена таким чином змінна буде видно тільки всередині селектора, де відбулося оголошення змінної. Для того щоб змінну можна було використовувати в будь-якому місці програми (оголосити змінну глобально) необхідно оголосити її всередині спеціального селектора :root. 

:root { --main-bg-color: teal; }

.one { background-color: var(--main-bg-color); }

.two { background-color: var(--main-bg-color); }

.three { background-color: var(--main-bg-color); }
