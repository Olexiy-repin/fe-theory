Технологія CSS

CSS (Cascading Style Sheets) - каскадні таблиці стилів — це мова, що використовується для опису оформлення документу, який написаний мовою розмітки HTML.

Якщо HTML використовується для визначення структури та семантики змісту документу, то CSS - для його стилізації та позиціювання. Каскадні таблиці стилів описують правила форматування елементів за допомогою властивостей і допустимих значень цих властивостей. Для кожного елементу можна використати обмежений набір властивостей, інші не впливатимуть на нього.

Синтаксис CSS

Синтаксис CSS дуже простий, оголошення стилю складається з двох частин: елементу вебсторінки — селектора і команди форматування — блоку оголошення. Спочатку пишеться ім'я селектора, наприклад body, це означає, що усі стильові параметри застосовуватимуться до тегу body, потім фігурні дужки, в яких записується стильова властивість, а її значення вказується після двокрапки.

CSS правило - це конструкція, яка зв'язує один або декілька селекторів з будь-якою кількістю властивостей CSS.

Селектор (selector) - це те, до чого ми хочемо застосувати css правило (tag, id, class тощо).

Блок оголошень (declaration block) - увесь вміст, що знаходиться між фігурними дужками після селектора.

Властивість (property) - ідентифікатор дії, яка застосовуватиметься до елементу. Наприклад, Ви хочете змінити зображення шрифту (font-weight), колір (color), розмір шрифту (font-size) тощо. Після імені властивості необхідно ставити двокрапку.

Значення (value) - описує, як саме властивість оброблятиметься браузером. Кожна властивість має набір допустимих значень, визначених формальними правилами, а також семантичний сенс. Записується після двокрапки.

Зовнішня таблиця стилів

Зовнішня таблиця стилів описується в зовнішньому файлі з розширенням .css, потім він підключається до документу за допомогою тегу link. Усередині файлу можуть міститися тільки стилі. До кожної вебсторінки можна приєднати декілька таблиць стилів, додаючи послідовно декілька тегів link. На сьогодні це стандарт підключення стилів до документу.

  • На одному рівні з index.html створюється папка css.
  • Усередині папки css створюється файл стилів, наприклад, styles.css.
  • У тегу head, використовуючи тег link, підключаємо створений раніше файл стилів. У атрибут href тегу link записуємо шлях до файлу стилів відносно index.html.

Види селекторів

Селектори становлять структуру вебсторінки. За допомогою них створюються правила для форматування елементів вебсторінки. Селекторами можуть бути елементи, їхні класи й ідентифікатори, а також псевдокласи та псевдоелементи.

Універсальний селектор - відповідає будь-якому HTML елементу.

Селектор елемента - описує, до якого типу елементу застосується css правило, властивості застосуються до всіх елементів даного типу.

Селектор класу - тегам можна вказувати спеціальний атрибут class. Ми можемо використати селектор класу, щоб застосувати стилі для одного чи декількох елементів з однаковим ім'ям класу, розміщених в різних місцях сторінки. У CSS перед ім'ям класу ставиться крапка .

Селектор ідентифікатора - тегам можна вказувати атрибут id. Він має бути унікальним на сторінці, це означає, що селектор id дозволяє форматувати один конкретний елемент. У CSS перед ідентифікатором ставиться #

Селектор нащадка - застосовує стилі до елементів, розташованих усередині елементу-контейнера. Якщо потрібно відформатувати нащадків певного елементу, йому треба вказати стильовий клас.

Дочірній селектор - дочірній елемент - прямий нащадок елементу, що містить його. У одного елементу може бути декілька дочірніх елементів, а батьківський елемент у кожного елементу лише один. Дочірній селектор застосує стилі, якщо між дочірнім і батьківським елементом нічого немає, тобто дочірній більше ні в що не вкладений.

Сестринський селектор - сестринські стосунки виникають між елементами, що мають спільного батька. Сестринські селектори дозволяють вибрати елементи з групи одного рівня.

Селектор атрибуту - вибирає елементи на основі імені чи значення атрибуту.

Селектор псевдокласу - псевдокласи — це стани елементу. Вони застосовують CSS правила, якщо виконується подія чи якщо елемент підкорюється певному правилу.

Наслідування, каскад, специфічність

Наслідування і каскад — два важливі поняття в CSS, які тісно пов'язані. Наслідування полягає в тому, що елементи наслідують властивості від батьківського елементу. Каскад відповідає за те, як різні види таблиць стилів застосовуються до документу, як правила, що конфліктують, перевизначають одне одного.

Наслідування

Наслідування - механізм, за допомогою якого певні властивості передаються від предка до його нащадків. Специфікацією CSS передбачено наслідування властивостей, що стосуються текстового вмісту сторінки: color, font, text-align тощо. Здебільшого це зручно, бо не потрібно вказувати розмір шрифту, сімейство шрифтів для кожного елементу вебсторінки.

  • Стилі можуть наслідуватися від батьківського елементу.
  • Стилі, розташовані в таблиці стилів нижче, скасовують стилі, розташовані вище.
  • До одного елементу можуть застосовуватися стилі з різних джерел. Перевірити, які стилі застосовуються, можна в інструментах розробника.
  • При визначенні стилю можна використовувати будь-яку комбінацію з таких селекторів: селектор елементу, псевдокласу елементу, класу або ідентифікатору елементу.

Властивості, що стосуються форматування блоків, не наслідують. Наприклад background, border, display, width и т.д.

Каскад

Каскадування — це механізм, який керує остаточним результатом у ситуації, коли до одного елементу застосовуються різні CSS правила. Є три критерії, які визначають порядок застосування властивостей: правило !important, специфічність і порядок підключення таблиць стилів.

  • До одного елементу можуть застосуватися декілька CSS правил. У такому разі властивості правил комбінуються.
  • Властивості елементу можна комбінувати явно за допомогою численних класів.
  • Коли в різних CSS правилах є однакові властивості з різними значеннями, то вони конфліктують.

Правило !important

Вагу правила можна вказати за допомогою ключового слова !important, яке додається відразу після значення властивості. Правило необхідно розміщувати в кінець оголошення перед дужкою, що закривається, без пропуску. Таке оголошення матиме пріоритет над усіма іншими правилами. Це правило дозволяє скасувати значення властивості та встановити нове для елементу з групи елементів у разі, коли немає прямого доступу до файлу із стилями.

Вкрай не рекомендується використовувати !important в сучасній розробці.

Специфічність

Для кожного правила браузер обчислює специфічність селектору. Якщо в елементу є оголошення властивостей, що конфліктують, то до уваги береться правило, що має найбільшу специфічність. Значення специфічності складається з чотирьох частин: 0, 0, 0, 0. Унаслідок до елементу застосуються ті правила, специфічність яких більша.

  • Специфічність визначається селектором.
  • Якщо специфічність однакова, то перемагає те правило, яке розташоване нижче в коді.
  • Що простіше селектори, то менше проблем з розрахунком специфічності.

Специфічність селектора визначається таким чином:

  • Кожен вбудований стиль, додає одиницю в перший розряд — 1, 0, 0, 0
  • Кожен id додає одиницю в другий розряд - 0, 1, 0, 0
  • Кожен class, атрибут, псевдокласс додає одиницю в третій розряд - 0, 0, 1, 0
  • Кожен елемент і псевдоелемент додає одиницю в четвертий розряд - 0, 0, 0, 1

Модель візуального форматування

Модель візуального форматування CSS є алгоритмом, який обробляє HTML документ і виводить його на екран пристрою. Ця модель перетворює кожен елемент документу таким чином, що він генерує нуль або більше прямокутних боксів відповідно до блокової моделі CSS.

Блокова модель

У блоковій моделі елемент розглядається як прямокутний контейнер, що має область вмісту і необов'язкові поля: рамки і відступи. Властивість display визначає тип контейнеру елементу. Для кожного елементу є значення браузеру за умовчанням.

Область вмісту (content) — це вміст елементу, наприклад, текст або зображення.

Внутрішнє поле (padding) — це відстань між основним вмістом і його межею (border). Якщо елементу вказати фон, то він пошириться і на поля елементу також. Внутрішній відступ не може мати від'ємні значення.

Зовнішній відступ (margin) - додає відступ ззовні елементу від зовнішньої межі рамки (border) до сусідніх елементів, у такий спосіб розділяючи елементи на сторінці. Зовнішні відступи завжди залишаються прозорими, через них видно фон батьківського елементу. Може мати від'ємні значення. Не обов'язковий, усталено значення рівне нулю.

Межа або рамка (border) - Якщо колір рамки не вказаний, вона приймає колір основного вмісту елементу, наприклад, тексту. Якщо рамка має розриви, то крізь них проступатиме фон елементу. Не обов'язкова, усталено значення дорівнює нулю.

Content-box усталено увімкнута у браузерах. Аналогічно розраховується висота.

Блокові та рядкові елементи

Блокові елементи

Блокові елементи (display: block) — елементи, які форматуються візуально як блоки, розташовуються на сторінці у вікні браузеру вертикально. Вони створюють розрив рядка перед елементом і після нього, утворюючи прямокутну область, яка по ширині займає усю ширину вебсторінки або блоку-батька (якщо для елементу не вказано значення width). Це елементи div, h1-h6, p, ol, ul, li та багато інших.

See the Pen core-module-3-block-el by GoIT ( @GoIT-fe) on CodePen.

Рядкові елементи

Рядкові (display: inline) - елементи, які генерують внутрішньо рядкові контейнери. Вони не формують нові блоки контенту.

Вони ігнорують верхні та нижні margin, padding, але якщо для елементу заданий фон, він поширюватиметься на верхній і нижній padding, вилазитиме на сусідні рядки тексту.

Ширина та висота рядкового елементу залежать тільки від його вмісту, не можна вказати розміри за допомогою CSS. Можна збільшити відстань між сусідніми елементами по горизонталі за допомогою горизонтальних полів і відступів.

See the Pen core-module-3-inline-el by GoIT ( @GoIT-fe) on CodePen.

Рядкові елементи можуть містити тільки дані й інші рядкові елементи. Виняток — елемент <a>, який згідно зі специфікацією HTML5 може обертати абзаци, списки, таблиці, заголовки та розділи, якщо вони не містять інших інтерактивних елементів — посилань і кнопок.

Рядково-блокові елементи

Рядково-блокові (display: inline-block) - такі елементи є вбудованим, але для них можна задавати поля, відступи, ширину і висоту.

Для того, щоб верхні та нижні поля і відступи працювали для рядкового елементу, треба використати display: inline-block. Елемент залишиться рядковим, але можна буде повноцінно застосовувати поля, відступи, вказувати ширину і висоту.

Перетворення типів елементів

Інколи необхідно, щоб рядковий елемент поводився як блоковий чи навпаки. Значення властивості display у елементу можна змінювати за потреби на будь-яке інше. Можна перетворити рядковий елемент у блоковий чи у рядково-блоковий або навпаки. Це стандартна практика при верстанні.

У нас усі елементи блокові (div), але content і sidebar потрібно поставити в один рядок, усталено з блоковими елементами це зробити неможливо, тому ми content і sidebar змінюємо display на inline-block. Потім указуємо їм якісь ширини (у прикладі ширини в % від загальної ширини), і ось вони вже стоять в одному рядку, бо тепер вони рядково-блокові. Якщо їх загальна ширина поміщається в один рядок, ми отримаємо бажаний результат.

See the Pen core-module-3-display-prop by GoIT ( @GoIT-fe) on CodePen.

Ознайомтеся з особливостями inline-block елементів, зокрема чому у них завжди є дивний відступ.

Особливості margin

Margin - зовнішній відступ блоку, використовується для візуальної реалізації вертикальних\горизонтальних проміжків між блоками. Він може бути в усіх елементів, окрім рядкових, яким можна вказувати лише горизонтальний margin.

Схлопування вертикальних відступів

Одне з найважливіших правил, які потрібно запам'ятати. Вертикальні відступи margin, що дотикаються, об'єднуються. При цьому ширина загального відступу дорівнює ширині більшого відступу. Злиття виконується тільки для блокових елементів в нормальному потоці документу.

Вкажемо верхньому блоку margin-bottom 30px, нижньому блоку margin-top 50px, остаточний margin не буде 80px, бо через схлопування відступів буде вибраний більший - 50px.

Випадання вертикальних відступів

Якщо всередині одного блоку розмістити інший блок і вказати йому margin-top, то внутрішній блок притискається до верхнього краю батьківського, а у батьківського елементу з'явиться відступ згори, тобто внутрішній блок "випаде" з батьківського блоку. Якщо у батьківського елементу також був заданий верхній відступ, то вибереться найбільше зі значень.

У прикладі ми вказали margin-top 50px для inner-box, відсунувся не він сам від верху outer-box, а увесь outer-box.

Верхній margin на стику з блоком-батьком неначе пробиває його і випадає назовні, це одна з найчастіших помилок при верстанні! Рекомендуємо бути дуже уважними та використовувати верхній і нижній margin для блоку-дитини в крайньому разі, якщо він вкладений у батьківський блок.

Фон

Незалежно чи журнал, картина або веб сторінка, фон у візуальному посланні завжди переслідує дві мети. Перша — це допомога в розпізнаванні, виділенні контенту. Якщо текст світлий, то фон має бути темним і навпаки. Інакше читання стає важким або неможливим. Друга мета використання фону — це допомога в сприйнятті сенсу, підказка про вміст контенту. Текст із назвою компанії на тлі пальм і морського пляжу підкаже сферу діяльності та допоможе прийняти рішення, чи продовжувати знайомство з контентом.

У браузері для роботи з фоном використовується властивість CSS background (і ряд інших властивостей, що розпочинаються з цього слова). Цих властивостей не багато, але вони відрізняються особливою моделлю поведінки та правилами розрахунку положення на екрані.

Фон батька і нащадків. Наслідування

Якщо елемент міститься в потоці документу (це відбувається усталено з усіма елементами), то фон нащадка буде над фоном батьківського елементу. Усталено всі елементи мають білий колір фону і властивість background не наслідується. Це означає, що фон завжди треба вказувати, якщо він відмінний від білого.

Фон для елементів, зокрема body, пов'язаний з їхньою висотою. Усталено висота цих елементів auto, тобто визначається контентом. Щоб заповнити екран фоном повністю і при цьому не залежати від кількості контенту, потрібно встановити для body властивість min-height: 100vh. Висота body буде мінімум по висоті viewport або більшою, якщо того потребуватиме контент.

Є можливість зробити колір фону прозорим або напівпрозорим. Це створює т.з. новий контекст, тобто змушує браузер враховувати додаткові чинники рендерингу сторінки: колір фону, border, padding і контент, що міститься під таким фоном. Це ускладнює розрахунки й вимагає додаткових ресурсів пристрою, примушує розробника витрачати більше часу на створення сторінки та перевірку зображення. Тому, якщо є вибір, то варто розглянути альтернативний варіант верстання.

Це короткий запис, який допускає опис до 5 різних властивостей фону одночасно. Розглянемо приклад цього короткого запису і потім перейдемо до знайомства з цими властивостями в повному записі.

background-size

У цього правила найбільша кількість значень. Найважливіше розуміти, як працюють два з них: cover і contain. Обидві властивості прагнуть максимально вмістити в контейнер фонове зображення, одна завдяки обрізанню зайвого, а інша внаслідок появи порожнього простору в контейнері. Можна вказувати числові значення. Якщо буде вказане одне значення, то браузер вважатиме його шириною фону і висоту обчислить сам, зберігаючи пропорції. Якщо будуть вказані два значення, то перше як завжди відповідатиме за горизонталь, тобто це буде ширина, а друге — висотою. І найцікавіше, якщо фону було задано декілька зображень, то через кому можна вказувати декілька наборів розмірів для цих зображень. Приклад.

Паралакс ефект з допомогою CSS

Паралакс (Parallax, зміна, чергування) - це зміна видимого положення об'єкту щодо віддаленого фону у залежності від розміщення спостерігача. Первинно цей термін вживався для природних явищ, в астрономії та геодезії.

У веб дизайні паралакс ефект або паралакс скролінг— це спеціальна техніка, коли фонове зображення в перспективі рухається повільніше, ніж елементи переднього плану. Ця технологія застосовується часто, бо має дійсно ефектний вигляд. Досягається такий ефект тривимірного простору з допомогою декількох шарів, які накладаються один на одного і при прокручуванні рухаються з різною швидкістю.

Основний мінус паралаксу — це проблеми з продуктивністю сайту. Усе має красивий і стильний вигляд, але застосування JavaScript, з допомогою якого створюється ефект паралаксу, значною мірою збільшує вагу сторінки та дуже погіршує швидкість завантаження. Це відбувається, бо в його основі лежать складні обчислення: JavaScript доводиться контролювати положення кожного пікселя на екрані.

Завдання трохи спростилося, коли з'явився CSS3. З його допомогою можна створити дуже схожий ефект, який буде економніше використовувати ресурси.
