Вступ

Веб-сайти — це основа інтернету. Здебільшого сайти створюються за допомогою трьох технологій:

  • HTML - відповідає за наповнення вебсторінки контентом, структуризацію інформації та виділення логічних блоків.
  • CSS - використовується для оформлення та позиціювання, представлення інформації на сторінці.
  • JavaScript - потрібен для додавання функціоналу, інтерактивної взаємодії з користувачем.

Поява HTML була обумовлена необхідністю створення простого засобу розмітки документів для полегшення навігації, яким могли б користуватися люди без спеціальних технічних знань.

Значна частина ресурсів в Інтернеті не відрізняється від ресурсів на вашому комп'ютері: це лише величезна кількість файлів і директорій.

У цьому курсі ми навчимося наповнювати HTML документ контентом і стилізувати його за допомогою CSS. Результатом будуть гарні та частково інтерактивні вебсторінки.

Операційна система та браузер

Перед початком вивчення основ написання коду необхідно підготувати робоче місце. Насамперед це стосується програмного забезпечення, що має бути встановленим та налаштованим, щоб робота у межах даного курсу була ефективною.

Розглянемо програмне забезпечення, яке вже встановлене на Ваш комп'ютер, це операційна система (ОС) і браузер.

  • На сьогодні найпоширеніші операційні системи: Windows, MacOS, Linux. Для Front - end розробки використовуються кросплатформні мови, тому операційна система на процес вивчення не впливатиме.
  • Браузер — програма, призначена для перегляду вебсторінок. Нам вона потрібна для перегляду результату верстання. Найпоширеніші браузери сьогодення: Google Chrome, Mozilla Firefox, Edge и Safari. У курсі ми використовуватимемо Google Chrome ,оскільки він найпопулярніший і чудово підходить для веброзробки.

HTML

HyperText Markup Language (мова розмітки гіпертексту) — стандартизована мова розмітки документів у Всесвітній павутині. Це набір правил, які використовуються для структуризації (розмітки) текстової інформації, додавання зовнішніх зображень, створення таблиць, списків тощо.

HTML-документ — це звичайний текстовий документ, який має розширення .html. Після інтерпретації цього документа браузером користувачі бачать не початковий код з елементами розмітки, а кінцевий результат обробки HTML документу.

Зараз Ви читаєте HTML документ, оскільки конспект - це вебсторінка, яка розмічена за допомогою HTML- синтаксису.

W3C

Популярність мови розмітки HTML спричинила необхідність управління стандартами. World Wide Web Consortium, або скорочено W3C - організація яка займається розвитком HTML і його стандартів.

Основним завданням консорціуму є забезпечення відкритості стандартів мережі Інтернет і її інтернаціоналізації. Відповідно до цієї ідеї уся документація по регулюванню принципів роботи мережі носить статус "рекомендацій" та є відкритою. Будь-який розробник має право на реалізацію стандартів W3C без обмежень.

Підхід відкритості та рекомендацій, а не зобов'язань є основною ідеєю розвитку мережі Інтернет. Проте відкритість стандартів має негативну сторону: відсутність зобов'язання розробників дотримуватися стандарту ускладнює реалізацію сайтів під браузери різних розробників.

Специфікація HTML

Специфікація HTML — фундаментальний, головний документ, що описує можливості та стандарти мови HTML.

Для верстальника важливі такі розділи, бо вони описують як теги можна вкладати один в одного і що означає кожен тег:

  • Семантика елементів
  • Типи контенту
  • HTML елементи

Категорії змісту

Виділяють 7 категорій контенту. Запам'ятовувати їх зараз не потрібно, але варто ознайомитися для кращого розуміння можливостей HTML.

  • Metadata content - метадані для браузерів, пошукових систем тощо (все, що в head, службова інформація).
  • Flow content - потоковий контент (все, що в body, те, що побачить користувач).
  • Sectioning content - великі смислові розділи документу.
  • Heading content - заголовки.
  • Phrasing content - фразовий контент (сам текст документу і дрібні текстові елементи, які дрібніші за абзац).
  • Embedded content - вбудовуваний контент (зображення, відео, аудіо тощо).
  • Interactive content - інтерактивні елементи (те, з чим взаємодіє користувач).

Теги

Основа HTML - це теги, які створюють різні елементи, вибудовують зміст і структуру кінцевого документа.

Тег (tag, іменована мітка, дескриптор) — елемент мови розмітки гіпертексту. Контент, що міститься між початковим і кінцевим тегом, зображається і розміщується відповідно до властивостей, вказаних в початковому тегу. Для виділення тегів серед іншого тексту документу, використовуються спеціальні символи - кутові дужки (angle brackets).

HTML документ складається з дерева HTML елементів і тексту. Кожен елемент позначається в початковому документі початковим і кінцевим тегом (за рідкісним винятком).

Початковий тег показує, де починається елемент, кінцевий - де закінчується. Кінцевий тег утворюється шляхом додавання слешу / перед ім'ям тега. Між початковим та кінцевим тегами знаходиться вміст тега - контент.

Теги можуть вкладатися один в одного. При вкладенні слід дотримуватися порядку їх закриття (принцип "матрьошки").

Алгоритм визначення можливості вкладення тега:

  • Йдемо в специфікацію, знаходимо розділ #4.
  • Перевіряємо контентну модель (Content model) тега, в який вкладаємо.
  • Перевіряємо категорії тега (Categories), який вкладаємо.
  • Якщо категорія підходить і обмеження не забороняють, то вкладати можна, інакше - не можна.

Теги можна розділити на дві категорії:

  • Парні теги - теги, що складаються з початкового (наприклад <p>) і кінцевого тега ( </p>). Парні теги огортають контент, дозволяючи змінювати його властивості або вид зображення. Усередину парних тегів можуть бути вкладені інші теги.
  • Одинарні теги - теги, що складаються тільки з початкового тега (наприклад <img>). Одинарні теги створюють один елемент на сторінці або служать для зміни властивостей документу, підключення файлів тощо. Вони отримують свій контент або поведінку з атрибутів.

Унаслідок огортання тексту між початковим та кінцевим тегами, він змінив своє зображення (став напівжирним, підкресленим або курсивним).

Є безліч тегів, які слугують для опису основних елементів вебсторінки і їх візуального оформлення. Умовно теги діляться на такі типи:

  • Теги верхнього рівня
  • Теги заголовку документу
  • Теги загального призначення

Не плутайте теги й елементи!

  • Теги — це структура мови HTML, вони вказують браузеру, який ми хочемо створити елемент.
  • Елементи — це те, з чого вже складається вебсторінка після відкриття HTML документу у браузері.

Теги верхнього рівня

Ці теги призначені для формування структури вебсторінки та визначають розділ заголовка і тіла документу.

html — кореневий елемент документу, контейнер, який містить у собі увесь вміст вебсторінки, включно з тегами head і body.

head — призначений для зберігання технічної інформації про сторінку: заголовок, опис, ключові слова для пошукових машин, кодування тощо. Уся ця інформація не зображається у вікні браузера, проте містить дані, які вказують браузеру як слід обробляти сторінку.

body — призначений для зберігання змісту вебсторінки, що показується у вікні браузеру. Контент, який повинен зображатися на сторінці, слід розташовувати саме усередині контейнера body. До такого контенту належать текст, зображення, таблиці, списки тощо.

У прикладі приведений набір тегів верхнього рівня та порядок їх вкладеності. Це основа будь-якого HTML документу, як фундамент для будинку.

Зверніть увагу на DOCTYPE перед тегом html. Цей обов'язковий елемент коду вебсторінки не є тегом, це оголошення типу документу. Він призначений для того, щоб повідомити браузер, в якій версії HTML написаний документ і як його інтерпретувати (в даному випадку це HTML5). Це гарантує, що сторінка матиме однаковий вигляд в усіх сучасних браузерах.

Теги заголовку документу

До цих тегів належать елементи, які містяться в head. Вони безпосередньо не показуються у вікні браузера, за винятком тега title, який визначає назву вебсторінки.

title

Обов'язковий тег. Текст, що міститься усередині нього, показується в рядку заголовка веббраузеру. Довжина заголовка має бути не більше 60 символів, щоб повністю поміститися в заголовку. Текст заголовка повинен містити максимально повний опис вмісту вебсторінки.

meta

Використовуються для зберігання інформації, призначеної для браузерів і пошукових систем: установки кодування документу, для передачі інформації пошуковим системам, для встановлення ширини та початкового масштабу для вікна перегляду на мобільних пристроях тощо. Наприклад, механізми пошукових систем звертаються до метатегів для отримання опису сайту, ключових слів і інших даних.

МетатегОпис
<meta charset="UTF-8">Документу задається кодування UTF - 8
<meta name="author" content="Alexander Repeta">Метатег, що визначає автора документу.
<meta name="keywords" content="HTML, тег, метатег">Метатег, що визначає ключові слова документу.
<meta name="description" content="Страница о тегах">Метатег, що визначає опис документу для пошукових систем.

Приклад

Використовуємо тег title для вказування заголовка і meta (з атрибутом charset, про атрибути ми поговоримо пізніше) для вказівки кодування сторінки.

Атрибути тегів

Атрибути дозволяють змінювати властивості та поведінку елементу, для якого вони задані. Загальна кількість атрибутів досить велика, але їх значення, як правило, можна згрупувати по різних типах, наприклад, ті, що задають розмір, адресу тощо.

Атрибути записуються усередині початкового тега, а їх значення містяться усередині лапок. Атрибутів може бути декілька. Деякі теги не можуть використовуватися без атрибутів.

Атрибути тега a:

  • href — вказує адресу сторінки, на яку перейде користувач, коли натисне на текст посилання.
  • target — вказує, в якому вікні відкриється нова сторінка.
  • title — додає до тексту посилання підказку, що спливає.

Атрибути тега img:

  • src — вказує на адресу зображення.
  • alt — вказує альтернативний текст, який буде показаний, якщо картинка не завантажиться.
  • width і height — задають ширину і висоту зображення.

Огляд основних тегів

HTML документ повинен мати зміст, як на сторінці з газети, у ньому є секції, заголовки, списки, зображення, абзаци текст тощо. Для усього різноманіття типів контенту є відповідні теги, які несуть в собі сенс, у такий спосіб описують контент. Розберім основні з них.

У кінці секції буде один великий приклад, що включає розглянуті теги.

Великі смислові розділи

article — незалежна, відокремлювана частина документу, що має значення поза його межами, наприклад твіт, стаття, віджет з рекламою. Бажаний заголовок.

section — смисловий розділ документу. Невідокремлюваний від основного документу, на відміну від article. Бажаний заголовок.

nav — навігаційний розділ з посиланнями на інші сторінки чи їх частини. Використовуйте для основної навігації, а не для усіх груп посилань в документі.

Заголовки <h1>...<h6>

Ця група тегів визначає текстові заголовки смислових розділів різного рівня, які показують відносну важливість секції, розташованої після заголовка.

Повинні використовуватися тільки для виділення заголовків нового розділу або підрозділу. При використанні заголовків необхідно враховувати їх ієрархію, тобто за h1 повинен слідувати h2 і так далі.

Зверніть увагу, що тег h1 використовується лише один раз як основний заголовок сторінки. На головних сторінках не завжди малюють заголовок першого рівня. Теги від h2 до h6, можуть використовуватися скільки завгодно, але їх завжди слід використовувати послідовно, вказуючи важливість і ієрархію заголовків контенту. Наприклад, h4 має бути підзаголовком h3, який має бути підзаголовком h2.

Потоковий вміст

header — ввідна частина смислового розділу або усього сайту, зазвичай містить підказки та навігацію. Часто використовується як шапка усієї сторінки або складної секції. Їх може бути декілька на сторінці.

main — основне, те, що не повторюється на інших сторінках. Може бути тільки один на сторінці.

footer — фінальна частина розділу або усього сайту, часто містить копірайт, список посилань на соціальні мережі, контактну інформацію тощо. Їх може бути декілька на сторінці.

Списки

Списки надають можливість упорядкувати, систематизувати колекції та представити їх в наочному і зручному для користувача вигляді. Кожен список є контейнером, усередині якого містяться елементи списку.

  • ol — створює нумерований список, тобто кожен елемент списку розпочинається з числа. Браузер нумерує елементи за порядком автоматично. Якщо видалити один або декілька елементів такого списку, то інші номери будуть автоматично перераховані.
  • ul — створює маркований список, кожен елемент якого розпочинається з невеликого символу - маркера.
  • li — визначає окремий елемент.

Параграф

Універсальний блоковий контейнер

Тимчасом як більшість HTML тегів визначають значення ( section виділяє секцію, h1 створює заголовок тощо), тег div взагалі не має ніякого семантичного значення.

Це може звучати дивно, але вони фактично використовуються для угрупування тегів для подальшої стилізації. До HTML5, div використовувався для угрупування логічно пов'язаного контенту, наразі у нас є семантичні теги.

Тег div належить до універсальних блокових контейнерів і застосовується в тих випадках, де потрібні блокові елементи без додаткових властивостей. У більшості ситуацій використовується як блок-обгортка для подальшої стилізації.

Як визначити, перед нами article, section або div?

  • Якщо ми можемо назвати розділ, винести його за межі нашого сайту і він матиме сенс - це article.
  • Якщо ми можемо назвати розділ і поза нашим сайтом він не матиме сенсу - це section.
  • Якщо ж ми не можемо дати чіткої назви групі контенту (не "права колонка", а щось змістовне) - це div і, найімовірніше, нам просто потрібне угрупування для стилізації.

Фразовий контент

Посилання

Якір — призначений для створення посилань. Всередину тега a поміщається текст, який показуватиметься на вебсторінці. Текст посилання зображається у браузері з підкресленням, колір шрифту - синій, за умови наведення на посилання, курсор миші змінює вигляд.

Атрибути:

  • href — обов'язковий атрибут, задає адресу вебсторінки або деякого ресурсу (містить абсолютний чи відносний шлях).
  • download="шлях до файлу" — повідомляє браузеру, що вказаний в href ресурс має бути завантажений в момент, коли користувач клацає по посиланню. Задаючи ім'я для атрибуту, ми вказуємо ім'я завантажуваному файлу. Можна використати атрибут без вказівки його значення.
  • target — вказує на те, в якому вікні повинен відкриватися документ, до якого веде посилання. Цікавим значенням можна виділити _blank, при якому сторінка відкривається в новому вікні.
  • rel — доповнює атрибут href інформацією про відносини між поточним і пов'язаним документом.

У посилань є можливість не лише переходити на інші сторінки та викачувати файли, але і здійснювати дзвінки на телефони, відправляти повідомлення або дзвонити по скайпу.

Контентне зображення

Використання графіки робить вебсторінки візуально привабливішими. Зображення допомагають краще передати суть і зміст документу. Тег img призначений для показу на вебсторінці зображень в різних графічних форматах. Не усі зображення на сторінці контентні, деякі фонові.

Атрибути:

  • src — обов'язковий, він вказує браузеру, де можна знайти зображення. Шлях до картинки може бути абсолютним чи відносним.
  • alt — є альтернативним описом. У разі, якщо картинка не завантажилася, цей атрибут необхідний для надання суттєвої інформації для користувачів, які не можуть бачити зображення (наприклад, якщо вони слабозрячі).
  • width — задає ширину зображення.
  • height — задає висоту зображення.

See the Pen module-1-img by GoIT ( @GoIT-fe) on CodePen.

Без вказання розмірів зображення малюнок показується на сторінці в реальному розмірі. Якщо буде заданий тільки один з атрибутів, то другий обчислюватиметься автоматично для збереження пропорцій малюнка.

Кнопка

Часто тег button це просто інтерактивний елемент, який "оживає" за допомогою JavaScript. Наприклад, кнопка закриття вікна, що спливає, або прокрутки каруселі.

Важливо не плутати посилання і кнопку. Якщо при кліці на елемент інтерфейсу, відбувається перехід за якоюсь адресою, тобто є href, то це посилання. Якщо ж при кліці, відбувається щось без перезавантаження або перенаправлення сторінки, то це кнопка.

span

На відміну від абзацу, span не розпочинається з нового рядка. Він рядковий, підкоряється правилу потоку документу "зліва направо" і займає ширину по своєму контенту. Призначений для визначення рядкового елементу усередині документу, не замінює тег p. Може використовуватися для угрупування і виділення рядкового контенту усередині абзацу для подальшої стилізації.

Блокові та рядкові елементи

Потік документу - порядок розташування елементів на сторінці. Потік документу стосується не смислової складової сторінки, а її зовнішнього вигляду (розташування елементів).

Є набір правил, яким підкоряються усі елементи, що містяться в потоці документу :

  • Вертикально потік документу протікає зверху вниз. Усталено, що раніше в коді розташований тег, тим вище він на сторінці. Типово елементи розташовуватимуться на сторінці в тому порядку, в якому вони вказані в HTML документі.
  • Горизонтально потік документу протікає зліва направо (справа наліво для східних країн). Елементи займають так звані рядки, як букви на листку в лінійку. У рядку елементи - зліва направо, коли ширини рядка бракує для вміщування утримуваного елементу, відбувається перенесення на наступний рядок.

Блокові елементи — характеризуються тим, що займають усю доступну ширину усередині батьківського блоку, висота елементу визначається його вмістом (якщо явно не вказана) і він завжди розпочинається з нового рядка. Візуально це прямокутні області, що йдуть один за одним зверху вниз.

Рядкові елементи — це елементи вебсторінки, які є безпосередньою частиною іншого елементу, наприклад, текстового абзацу. Переважно вони використовуються для зміни вигляду тексту або його логічного виділення. Займають по ширині і висоті розміри відповідно до свого контенту і вибудовуються в один рядок, якщо в рядку вистачає місця, інакше перестрибують на наступний і заповнюють його.

See the Pen block vs inline by GoIT ( @GoIT-fe) on CodePen.

Завжди під час верстання думайте про тип створюваного елементу: це допоможе уникнути грубих помилок і зробить процес верстання більш усвідомленим. Дізнатися тип тега Ви можете в специфікації або довідниках.

Абсолютні та відносні шляхи

Вебсайти містять безліч документів, які розміщують в окремі папки для зручного використання. Щоб створити посилання на файл, що знаходиться поза папкою, що містить поточний документ, необхідно вказати розташування файлу або шлях. Є два види шляхів до файлів і папок.

Абсолютний шлях

Вказує точне місце розташування файлу в структурі папок у проєкті або на сервері. Абсолютний шлях до файлу дозволяє отримати доступ до файлу зі сторонніх ресурсів і має вигляд https://images.pexels.com/photos/583842/pexels-photo-583842.jpeg

Відносний шлях

Описує шлях до вказаного документу відносно поточного. Відносні посилання використовуються при створенні посилань на інші документи на одному і тому ж сайті. Коли браузер не знаходить в посиланні протокол, він виконує пошук вказаного документу на тому ж сервері. Відносний шлях складається з імені папки, що вказує шлях до файлу, і імені самого файлу.

Для складання шляху відносних посилань є декілька спеціальних позначень:

  • / - вказує на кореневу папку, наприклад, /img/smile.png
  • ./ - вказує на поточну папку, наприклад, ./img/smile.png
  • ../ - піднятися на одну папку вище, наприклад, ../img/smile.png

Стаття, яку варто прочитати

Коли в шляху вказується тільки ім'я файлу, браузер припускає, що файл знаходиться в тій же папці, що і документ, що містить посилання.

Особливості графічних форматів

Графічний файл — це опис кольору кожного пікселя в заданому прямокутнику, цей прямокутник також називають розміром зображення. Наприклад, зображення розміром 800 на 600 (йдеться про пікселі).

  • Растрова графіка — опис графічного файлу як масиву з координатами кожного пікселя та описом їхніх кольорів. Основні представники: jpeg, png, webP, gif.
  • Векторна графіка — описується як правила або рівняння, що визначають лінії. Також додаткові властивості: колір ліній і фон для фігур, що утворюються замкнутими лініями. Це формат svg.

Векторна графіка зображатиметься однаково добре на звичайних екранах і на екранах з високою піксельною щільністю. Растрова графіка, особливо стисла з втратою якості, матиме розмитий вигляд.

JPEG — великі файли не вимагають прозорого фону або анімації. Зображення JPEG ідеальні для фотографій: вони можуть містити мільйони різних кольорів.

PNG — на відміну від JPEG, має додатковий параметр для опису прозорості - альфа-канал. Цей формат пасує для зображень зі складною формою, яка не вписується в прямокутник. У такому разі JPEG матиме білий фон, що не завжди збігається з кольором фону підкладки. Широко використовується для іконок соціальних мереж і графічних елементів навігації.

webP — попри вбудовану підтримку лише в Google Chrome, Opera й Android, варто враховувати, що йдеться про більш ніж 80% ринку браузерів. За умови однакової якості, стислі зображення будуть менше PNG на 26%, для JPEG розмір файлу webP буде меншим на 25-34%. Долучання альфа-каналу "додасть" лише 22%. Google, Facebook і eBay використовують цей формат для зберігання зображень на своїх серверах.

SVG — ідеально підходить для простих масштабованих зображень, оскільки займає дуже мало місця. За умови зростання складності зображення, розмір файлу збільшується - формат перестає бути вигідним. Як правило, цей формат використовується для логотипу, іконок і елементів графічної навігації.

Символьні підстановки

Під час верстки можна забути про одну важливу специфіку HTML, яка веде до помилок розмітки. Використання в тексті символів <, / і ", які можуть бути сприйняті браузером як керуючі структури, це спецсимволи (знаки більше і менше, слеші, лапки і т.д.).

Спецсимволи чутливі до регістру, тому їх необхідно прописувати точно так, як вказано в таблиці. Спецсимволи, що не мають мнемоніки, можуть не відображатися зовсім або ж некоректно відображатися в тих чи інших браузерах.

При використанні даних символів в тексті документа, браузер може порахувати їх частиною тегів і закрити або згенерувати тег, якого немає в коді, що призводить до помилок у верстці. Щоб цього уникнути спецсимволи вставляють в HTML у вигляді набору кодів. Розрізняють три варіанти додавання символів:

  • &мнемокод; – найбільш простий в запам'ятовуванні (наприклад - ©)
  • &#КОД10; - вставка спецсимволи за допомогою десятеричного коду ( приклад - &)
  • &#КОД16; - вставка спецсимволи за допомогою шістнадцятирічного коду ( приклад - ®)

Як видно, всі послідовності починаються з символу & (амперсанд) і закінчуються символом ; (крапка з комою).

Правила оформлення коду

Оформлення HTML і CSS документів - це важливе завдання. Насамперед, важливе для розробника, бо в процесі роботи над створенням вебресурсів досить часто необхідно повертатися до вже написаного коду. Здебільшого на проектах більше одного розробника. Якщо код не оформлений, він втрачає читабельність, швидкість розробки падає.

Цей курс вимагає якісного оформлення коду, збережіть цей Style Guide і дотримуйтеся його.

У HTML коментар огорнутий спеціальними роздільниками, текст між ними не показуватиметься.

Інструменти розробника

Ефективна робота неможлива без спеціальних знарядь. Для початкової розробки вебсторінок або навіть невеликого сайту необхідно навчитися працювати з інструментами розробника.

Вони є в усіх сучасних браузерах і не потребують додаткової установки.

Chrome DevTools

Інструменти розробника — функціонал браузеру, який дозволяє отримувати спеціалізовану інформацію про початковий код сайту і процес роботи у браузері.

Інструменти розробника можна викликати такими методами:

  • Натиснувши клавішу F12
  • Комбінацію Ctrl+Shift+I для Windows
  • Комбінацію Cmd+Opt+I для Mac
  • Навести курсор миші на елемент вебсторінки, що цікавить, натиснути праву клавішу і вибрати пункт меню "Проглянути код".

Усталено в інструментах розробника відкривається вкладка Elements. У цій вкладці представлена внутрішня HTML структура документу. За умови наведення курсору миші на певний елемент - він підсвічуватиметься в робочому вікні браузеру. У такий спосіб можна зручно та швидко вивчити поточну структуру HTML документу.

Окрім інспекції елементів, інструменти розробника дозволяють робити в HTML коді зміни, які відразу зображуватимуться в робочому вікні браузеру, але не зберігатимуться. У такий спосіб дуже зручно експериментувати з HTML документами з метою їх подальшої зміни.

  • Повний огляд DevTools
  • Відео про роботу з HTML деревом в DevTools (рекомендується виконати завдання після відео)
  • Відео по роботі з CSS в DevTools (рекомендується повернутися до цього відео в 3-му модулі)

Валідація HTML документа

Валідація HTML документа призначена для виявлення помилок у синтаксисі вебсторінки та розбіжностей зі специфікацією HTML. Відповідно програма або система для такої перевірки називається валідатором. За допомогою валідації документу розробник отримує список рекомендованих виправлень.

У процесі роботи валідатор перевіряє такі пункти:

  • правильність написання тегів
  • актуальність тегів і можливість їхнього
  • застосування в цій версії HTML
  • правильність закриття і вкладення тегів
  • правильність використання атрибутів

Є спеціальний сервіс, який перевіряє код HTML документу на відповідність стандарту.

Відмінність помилок від попереджень:

  • Помилки позначаються як Error. Їх треба обов'язково виправляти.
  • Попередження позначаються як Warning. Виправляти їх не обов'язково, але це привід замислитися.

Зверніть увагу, якщо Ви помістите у вікні валідатора просто окремі HTML теги, буде помилка. Валідатор перевіряє переданий йому HTML код на відповідність стандартам кодування HTML сторінки. Згідно зі стандартом у кожної сторінки має бути тег, який визначає тип стандарту ( doctype), який має бути застосований до коду відповідної сторінки. Також у кожного документу обов'язково мають бути теги: html, head, title, body

Основні переваги валідного HTML

Упевненість в завтрашньому дні — сайт, який строго дотримується вебстандартів, зводить до мінімуму імовірність, що з виходом нових версій браузерів треба буде залучати розробників для адаптації сайту. Хоча, звичайно, 100% гарантій немає.

Оптимізація командної роботи — командна робота дуже важка, а іноді й просто неможлива, якщо немає чітко визначених правил і стандартів розробки. Валідний код дає можливість швидко включиться в роботу розробникові, попри те, що до нього з цим сайтом працювало багато людей.

Мінімізація витрачених ресурсів на усунення багів — якщо розвалився сайт, насамперед більшість верстальників перевіряє валідність коду. Якщо сайт міститиме дві сотні помилок, нехай навіть не критичних, то, щоб серед них знайти ту, яка ламає сайт, може знадобитися багато часу, за який потрібно буде заплатити. Декілька годин простою сайту можуть призвести до втрати багатьох клієнтів і доходів від реклами.

Краще SEO — більшість професійних SEO- оптимізаторів переконані, що валідний код позитивно впливає на рейтинг сайту для пошукових систем. Вони намагаються видавати якісні сайти користувачам, а валідація це один із критеріїв якості.
