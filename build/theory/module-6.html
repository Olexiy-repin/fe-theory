Технологія Flexbox

Зараз найбільш популярною технологією позиціювання елементів є Flexbox. У неї хороша підтримка браузерів, на сьогодні Flexbox - це стандарт.

Ідея технології полягає в тому, щоб розділити елементи на блок-контейнер і керовані елементи — безпосередні нащадки контейнера. Блок-контейнер визначає загальний вигляд і напрям угрупування елементів (вирівнювання в колонку або ряд, порядок появи елементів і відстань між ними), а елементи мають можливість вказувати унікальні характеристики (встати на позицію іншого елементу, змінивши заданий в html порядок). Для реалізації такої гнучкості в технологію введені нові абстракції — напрями, уздовж яких вибудовуються елементи, лінії меж блоків і елементів тощо.

See the Pen flex-block-item by GoIT ( @GoIT-fe) on CodePen.

Текст як елемент flex-container. Кожен дочірній елемент flex-контейнера стає flex-елементом. Текст, який безпосередньо міститься в flex-контейнері, обертається анонімним flex-елементом. Текст, що складається з пробільних символів, поводиться так, немов набуває властивості display: none.

Axis

Елементи flex завжди розташовуються уздовж головної осі. Головна вісь може бути розташована горизонтально (row) або вертикально (column).

  • Головна вісь завжди перпендикулярна допоміжній осі.
  • Напрям головної осі можна змінити.
  • Порядок розташування елементів уздовж головної осі можна змінити.
  • Це не вплине на напрям допоміжної осі.

Приклад показує збереження напряму допоміжної осі при зміні розташування елементів уздовж головної осі. Слово start завжди притиснуте в даному прикладі до початку допоміжної осі (top або left) незалежно від прямого чи реверсного зображення елементів уздовж головної осі.

Властивості flex-елементу

Потужність flex полягає в можливості управління положенням окремого елементу і навіть у зміні порядку розташування елементу без будь-якого втручання в код HTML.

order

Це можливість управління положенням окремого елементу. Елементи розташовуються в порядку зростання значення order. Якщо однакові значення order порядок визначається послідовністю розташування елементів в HTML. Усталено в усіх елементів order: 0.

See the Pen flex-order by GoIT ( @GoIT-fe) on CodePen.

Властивість order впливає тільки на візуальне зображення. У системах, що озвучують вміст сайту, ця властивість не працює.

flex-grow

Визначає пропорцію значення width елементу flex контейнера, яка вирахується шляхом видалення вільного простору. Усталено це значення для усіх елементів дорівнює 0. Від'ємні значення не підтримуються.

У прикладі спостерігаємо, якщо значення 1 - усі елементи мають рівну ширину. Якщо однакові пропорції, значення ширини елементів рівні, хоча і мають різне значення: flex-grow - 2 для середнього рядка і 3 для нижнього. Grow - це "зростання", що вище значення flex-grow - то більше ширини елементу.

See the Pen flex-grow by GoIT ( @GoIT-fe) on CodePen.

flex-shrink

Визначає пропорцію значення width елементу flex контейнера, яка буде вирахувана, використовуючи ширини інших елементів (тобто за відсутності вільного простору). Усталено це значення для усіх елементів дорівнює 1. Від'ємні значення не підтримуються.

У прикладі спстерігаємо, якщо значенні 1 - усі елементи мають рівну ширину. Якщо однакові пропорції, значення ширини елементів рівні, хоча мають різне значення: flex-grow - 2 для середнього рядка і 3 для нижнього. Shrink - це "зменшення", що вище значення flex-shrink - то менше ширини елементу.

See the Pen flex-shrink by GoIT ( @GoIT-fe) on CodePen.

flex-basis

Визначає ширину елементу до застосування flex-grow і flex-shrink. Тобто до розподілу простору flex-контейнера. Допустимі стандартні значення розмірів в px, rem, % тощо. Очікується підтримка додаткових значень цієї властивості : content, max-content, min-content, fit-content і тому подібне.

flex

Коротка форма запису для перелічених вище flex-grow, flex-shrink і flex-basis. Обов'язковим для оголошення є тільки flex-grow.

align-self

Властивість, що дозволяє змінити правило align-items для окремого елементу. На жаль, доступні лише два значення: flex-start і flex-end.

CSS-тіні

Тінь під текстом або елементом зазвичай застосовується для створення ефекту тривимірності, залучення уваги до елементу або як частина дизайну. Невелика тінь під елементами надає також сторінці обсяг і глибину.

text-shadow

Властивість text-shadow використовується для додавання тіні до тексту. Тінь тексту дозволяє створювати дивовижні ефекти. Тіні можуть бути одношаровими або багатошаровими, розмитими, кольоровими або напівпрозорими.

Кожна тінь застосовується як до самого тексту, так і до елементів його оформлення. Одночасно можна задавати декілька тіней, вказуючи їх через кому. Тіні накладаються один на одного, не перекриваючи сам текст. Перша тінь завжди розташована зверху над іншими тінями. Властивість text-shadow успадковується.

Кожна тінь визначається двома або трьома значеннями і кольором. Якщо колір тіні не вказано, використовується колір тексту.

text-shadow: x-offset y-offset blur color;
  • x-offset - зміщення тіні по горизонталі щодо тексту. Може приймати як позитивні, так і негативні значення. Обов'язкове значення.
  • y-offset - зміщення тіні по вертикалі щодо тексту. Може приймати як позитивні, так і негативні значення. Обов'язкове значення.
  • blur - визначає радіус розмиття тіні. Чим більше радіус, тим сильніше розмита тінь. Необов'язкове значення.
  • color - за замовчуванням приймає колір тексту. Для значення можна використовувати hex, rgb, rgba формати запису кольору. Необов'язкове значення.

Якщо значення довжин негативні, то тінь зміщується вліво і вгору від тексту.

box-shadow

Властивість box-shadow ддодає елементу одну або більше тіней. Тіні бувають зовнішні або внутрішні, розмиті або плоскі, вони можуть слідувати контурам блоків із закругленими кутами. За допомогою ключового слова inset створюються тіні всередині елемента, роблячи елемент візуально об'ємним або утисненим.

Кожна тінь приймає від одного до п'яти параметрів:

box-shadow: inset x-offset y-offset blur spread color;
  • x-offset - зміщення тіні по горизонталі щодо блоку. Може приймати як позитивні, так і негативні значення, позитивне зміщує тінь вправо від блоку, негативне - вліво. Обов'язкове значення.
  • y-offset - зміщення тіні по вертикалі щодо блоку. Може приймати як позитивні, так і негативні значення, позитивне зміщує тінь вниз, негативне - вгору. Обов'язкове значення.
  • blur - визначає радіус розмиття тіні. Чим більше радіус, тим сильніше розмита тінь. Можна використовувати тільки позитивні значення. Необов'язкове значення.
  • spread - розширює тінь, потовщуючи її. Приймає як позитивні, так і негативні значення. Необов'язкове значення.
  • color - необов'язкове значення. За замовчуванням тінь чорного кольору. Для встановлення значення можна використовувати hex, rgb, rgba формати запису кольору.
  • inset - створює тінь всередині блоку.

CSS-трансформації

CSS-трансформації дозволяють змінювати розмір, форму і положення елемента на веб-сторінці за допомогою властивості transform.

Трансформації перетворюють елемент так, що інші елементи не зрушуються щодо нього. За замовчуванням трансформація відбувається відносно центру елемента.

Найбільш часто використовують коротку форму запису. Коротка форма еквівалентна запису в дужках - де змінюється горизонтальна і вертикальна координати.

Важливо запам'ятати, що при послідовному застосуванні цих значень зміна послідовності може привести до іншого результату. Причина - точка відліку перетворення. Ця точка відліку зміщується при кожному перетворенні.

Translate

Переміщує елемент в горизонтальному і / або вертикальному напрямках.

translate (translateX, translateY), translate3d, translateZ

transform-origin

Дозволяє змістити центр трансформації, щодо якого відбувається зміна положення / розміру / форми елемента. Значення за замовчуванням — center, або 50% 50%. Задається лише для трансформованих елементів. Чи не успадковується.

CSS-переходи

CSS-переходи дозволяють анімувати початкове значення властивості на нове значення з плином часу, керуючи швидкістю зміни значень властивостей.

Зміна властивостей відбувається при настанні певної події, яке описується відповідним псевдоклас, наприклад :hover.

Властивості transition це короткий запис чотирьох властивостей:

ВластивістьОпис
transition-propertyМістить назву CSS-властивостей, до яких буде застосовано ефект переходу. Значення властивості може містити як одну властивість, так і список властивостей через кому.
transition-durationЗадає проміжок часу, протягом якого повинен здійснюватися перехід. Якщо різні властивості мають різні значення для переходу, вони вказуються через кому. Якщо тривалість переходу не вказана, то анімація при зміні значень властивостей відбуватися не буде.
transition-timing-functionВластивість задає тимчасову функцію, яка описує швидкість переходу об'єкта від одного значення до іншого. Здається функцією Безьє.
transition-delayЗатримка, після якої почнеться анімація.
transition-property: color;
transition-duration: 1.2s;
transition-timing-function: cubic-bezier(.20, .96, .74, .07);
transition-delay: .5s;

Розглянемо приклад зміни декількох властивостей.

У transition-property для цих властивостей задані початкові значення. При спрацьовуванні псевдокласу hover і active в перерахуванні transition-property вказані властивості, значення яких треба анімувати і їх кінцеві значення. В обох випадках тривалість анімації складе 2 секунди.

See the Pen module-7-animation by GoIT ( @GoIT-fe) on CodePen.

Короткий запис переходу

Всі властивості, що відповідають за зміну зовнішнього вигляду елемента, можна об'єднати в одну властивість transition.

transition: transition-property transition-duration transition-timing-function transition-delay;

Переходи застосовуються не до всіх властивостей і їх значень. З повним переліком можна ознайомитися на MDN.

CSS-анімація

Як і переходи, анімація надає сайтам динамічність, але на відміну від переходів, анімації не потрібен обов'язковий ініціатор і між початковим і кінцевим станами можливо багато проміжних станів.

Створення анімації базується на ключових кадрах, які дозволяють автоматично відтворювати і повторювати ефекти протягом заданого часу, а також зупиняти анімацію.

Анімація реалізована властивістю animation. Не всі властивості (але більшість) елемента можуть бути анімовані. Повний список властивостей, які анімувати можна, знаходиться на MDN

Створення анімації

Створюється анімація шляхом установки ключових кадрів - @keyframes, мінімум два ключових кадру. Кадри визначають, які властивості на якому етапі будуть анімовані. Кожен кадр може включати один або більше блоків оголошення з одного або більше пар властивостей і значень.

Ключові кадри створюються за допомогою ключових слів from і to ( 0% і 100% ) або за допомогою відсотків.

/* Ім'я повинно бути описовим */
  @keyframes animationName {
    from {
      /* Тут перераховуються CSS-свойства */
      opacity: 0;
    }
    to {
      /* Тут перераховуються CSS-свойства */
      opacity: 1;
    }
  }

Застосування анімації

  • Якщо просто додати анімацію до стилю, який тут же застосовується до елементу, то анімація буде застосована при завантаженні сторінки.
  • Анімацію можна застосувати до одного з псевдокласів, включаючи: :hover , :active , або :focus.
  • Анімацію можна застосувати до стилю класу, і скористатися JavaScript.
  • До одного і того ж елементу можна застосувати більш однієї анімації, записавши їх через пробіл.
  • Види анімацій можна перераховувати через кому.
@keyframes animationName {
  from { opacity: 0; }
  to { opacity: 1; }
}

.className {
  animation-name: animationName;
  animation-duration: 1s;
}

Короткий запис властивості animation

Значення властивостей потрібно перераховувати в такому порядку

  1. animation-name - ім'я анімації, рекомендується використовувати назву, що відображає суть анімації. Обов'язково вказувати.
  2. animation-duration - встановлює тривалість анімації, задається в секундах або мілісекундах, негативні значення не припустимі. Обов'язково вказувати.
  3. animation-timing-function - визначає зміну швидкості від початку до кінця анімації за допомогою тимчасових функцій.
  4. animation-iteration-count - дозволяє запустити анімацію кілька разів.
  5. animation-direction - задає напрямок повтору анімації.
  6. animation-delay - задає затримку анімації.
  7. animation-fill-mode - визначає порядок застосування визначених у @keyframes стилів до об'єкта.
animation: animation-name animation-duration animation-timing-function animation-delay animation-iteration-count animation-direction;
.fade {
  animation: fadeOut 2s ease-in-out 2 alternate 5s forwards;
}

/* Соответсвует */

.fade {
  animation-name: fadeOut;
  animation-duration: 2s;
  animation-timing-function: ease-in-out;
  animation-iteration-count: 2;
  animation-direction: alternate;
  animation-delay: 5s;
  animation-fill-mode: forwards;
}

Пауза анімації

.fade:hover {
  animation-play-state: paused;
}
