Модуль 2 - Тесты

  1. Зачем нужна система контроля версий?

    1. Всё вышеперчисленное.
    2. Для организации совместной разработки.
    3. Для недопущения потери кода.
    4. Для возможности контроля над процессом разработки.

    Правильный ответ: 1

  2. Какова основная функция локального репозитория?

    1. Хранение сохраненных версий файлов. Возможность восстановления всех сохраненных изменеий.
    2. Автоматическая фиксация изменений прогресса работы.
    3. Хранение последней версии файлов проекта.
    4. Хранение только изменений в файлах проекта.

    Правильный ответ: 1

  3. В чем отличие удаленного репозитория от локального?

    1. Удаленный репозиторий предоставляет возможность совместной работы над проектом.
    2. Удаленный репозиторий хранит последние версии файлов проекта.
    3. Локальный репозиторий хранит последнюю версию файлов проекта, удаленный хранит все версии файлов проекта.
    4. Локальный репозиторий хранит все версии файлов проекта, удаленный хранит последнюю версию файлов проекта.

    Правильный ответ: 1

  4. Что делает команда git status?

    1. Показывает состояние файлов рабочей директории и промежуточной области.
    2. Показывает состояние истории коммитов проекта.
    3. Показывает состояние удаленного репозиторий.

    Правильный ответ: 1

  5. Что делает команда git add?

    1. Добавляет изменения из рабочей директории в промежуточную область (stage).
    2. Добавляет изменения из рабочей директории в историю проекта.
    3. Добавляет измененные файлы в удаленный репозиторий.
    4. Добавляет измененные файлы в локальный репозиторий.

    Правильный ответ: 1

  6. Что делает команда git commit?

    1. Фиксация изменений в коде, и их сохранение в истории локального репозитория.
    2. Добавление комментария к изменениям.
    3. Сохранение изменений в удаленном репозитории.
    4. Получение изменений от удаленного репозитория.

    Правильный ответ: 1

  7. Что делает команда git push?

    1. Отправляет локальную историю коммитов на удаленный репозиторий.
    2. Добавляет файлы в локальный репозиторий.
    3. Забирает историю коммитов с удаленного репозитория.
    4. Отменяет изменения в истории проекта.

    Правильный ответ: 1

  8. Зачем используется файл .gitignore

    1. Для указания файлов и папок которые запрещены к добавлению в репозиторий.
    2. Для указания технических параметров репозитория.
    3. Для указания файлов в которых запрещены изменения.
    4. Для указания данных связи локального и удаленого репозиториев.

    Правильный ответ: 1
